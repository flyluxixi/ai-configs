# d-prompt

把用户的图像需求，优化成一段可直接粘贴到 **ChatGPT / gpt-image-2 / DALL-E** 的高质量自然语言 prompt。**只产 prompt，不调用任何生图端点**——零外部依赖，优化全程由会话 AI 自身推理完成。

定位与边界：
- 本 skill 的产物是**提示词**，不是图片。用户拿这段 prompt 自己去任意生图工具出图。要本 skill 直接生图，用 **d-image-2**（它内部也会优化 prompt 再生图）。
- 主目标模型：**`gpt-image-2`**（含 `gpt-image-2-2026-04-21`；ChatGPT 网页版 2026 年默认底模），向下兼容 `gpt-image-1` / DALL-E 3。统一风格：**自然语言完整句式**，不堆逗号 tag 串、不带 `--ar`/`--v` 等参数（那是 Midjourney / SD 的语法，不在本 skill 范围）。
- 主用途是界面 UI 概念稿优化；非 UI 需求（人像/风景/插画/海报等）走轻量通用兜底。

## 工作流

1. **读方法论**：先读取本 skill 目录下的 `PROMPTING.md`，按其流程执行。

2. **适用判断**：按 `PROMPTING.md` 判断是 UI 需求还是通用图需求；用户说「用我的原话 / 不要优化」→ 跳过优化，按其原意直翻直传。

3. **（仅 UI 路径，最先做）继承项目视觉锚点**：生成前先查项目是否已定义视觉语言——约定文件 `docs/visual-language.md`（git 根下），或用户当场给。
   - **有 → 直接继承**：把北极星产品 + 核心 tokens（主色/中性/语义色/圆角/密度/字体/阴影/**装饰档位**）全带进 prompt，**不再重选 style**；此时风格锚点维度已满足，跳过风格追问，在推断项注明「已继承项目视觉锚点」。
   - **无 → 临时选一个 + 推回正确层**：用产品锚点库选合理默认，并**明确提示用户**：本次只是临时锚点；要真正约束前端，请把视觉语言固化进 `docs/visual-language.md`，再落进工程 `design-system.md` 与组件库主题（如 uview-plus / shadcn）。d-prompt 只**消费**视觉语言，不是它的权威来源。

4. **评估关键维度 + 交互式追问**：平台 / 页面类型 / 风格锚点 / **装饰档位（L0~L4）** 是否齐全？
   - 齐全 / 已继承锚点 / 已给参考图 → **不追问**直接优化（参考图走「参考图驱动」：先读图提取风格 + 装饰档位再套用）。
   - 缺风格锚点 → 用知名产品对标当选项**一次性问完**，不来回挤牙膏。
   - 装饰档位无法从「风格锚点 + 页面类型」推断（如"App 首页"既可能 L1 工具感、也可能 L3 消费感）→ 一并追问：克制后台感（L0）/ 工具感（L1）/ AI 感（L2）/ 消费 App 感（L3）/ 营销 Landing 感（L4）。
   - 具体追问策略与档位推断规则见 `PROMPTING.md` ②。

5. **优化**：按 `PROMPTING.md` 的优化维度（5 个**高权重维度**优先：Visual Language / **Visual Decoration** / Layout Structure / Content Realism / Design System）+ 产品锚点库 + 英文骨架，把需求补全为高质量 prompt。**英文骨架尾段按维度 4 选定的装饰档位二选一**：L0~L1 用「A 强 negative」模板（含 not glassmorphism / not gradient-heavy hero / not decorative imagery）；L2~L4 用「B 软化 negative」模板（去掉对应 negative，只保留"装饰应服务内容、不喧宾夺主"）。**整段照抄不要漏**，对 gpt-image-2 减少跑偏的收益最大。

6. **输出契约**（展示给用户过目，等确认或微调）：
   1. **中文版 prompt**（自然语言长句，便于核对语义）。
   2. **英文版 prompt**（自然语言长句，gpt-image-2 / DALL-E 英文更稳，推荐实际使用这版）。
   3. **推荐画布尺寸 / 比例 + 理由**（按平台映射；纯信息性，用户自行在目标生图工具里设置，本 skill 不绑定任何参数）。
   4. **推断项清单**（风格/**装饰档位**/配色/组件等用户没明说、你替他补的，逐条列出可否决；走参考图驱动时额外列出从参考图提取的风格特征 + 推定档位供用户确认）。
   - **载体格式（强制）**：两份 prompt 各自用 fenced code block（` ``` ` 三反引号）整段包裹，**不要用引用块 / blockquote（`>`）**——prompt 是要被用户整段复制粘贴到 ChatGPT 的，代码块才有一键复制、且复制出来不掺 `>` 等格式符号；引用块在终端 CLI 里每行带前缀，没法整段干净复制。尺寸、推断项等说明性内容正常排版即可，只有 prompt 本体进代码块。

7. 用户确认后即结束——交付这段 prompt 即为完成，不生图、不写文件（除非用户明确要求把 prompt 存成文件）。

## 与其他 skill 的边界

- **要图片** → d-image-2（它内部也会优化 prompt 再生图）。
- **只要提示词** → 本 skill。
- 本 skill 产物可直接喂给 d-image-2 的 `--prompt`，也可粘贴到 ChatGPT 网页版 / 其他 gpt-image 接口。
