# d-prompt

把用户的图像需求，优化成一段可直接粘贴到 **ChatGPT / gpt-image / DALL-E** 的高质量自然语言 prompt。**只产 prompt，不调用任何生图端点**——零外部依赖，优化全程由会话 AI 自身推理完成。

定位与边界：
- 本 skill 的产物是**提示词**，不是图片。用户拿这段 prompt 自己去任意生图工具出图。要本 skill 直接生图，用 d-image-2。
- 目标模型锁定 gpt-image / DALL-E 系：**自然语言完整句式**，不堆逗号 tag 串、不带 `--ar`/`--v` 之类参数（那是 Midjourney / SD 的语法，不在本 skill 范围）。
- 主用途是界面 UI 概念稿优化；非 UI 需求（人像/风景/插画/海报等）走轻量通用兜底。

## 工作流

1. **读方法论**：先读取本 skill 目录下的 `PROMPTING.md`，按其流程执行。
2. **适用判断 + 需求评估**：按 `PROMPTING.md` 判断是 UI 需求还是通用需求；再评估关键维度（平台 / 页面类型 / 风格锚点）是否齐全。
   - **（仅 UI 路径，最先做）继承项目视觉锚点**：生成前先查项目是否已定义视觉语言——约定文件 `docs/visual-language.md`（git 根下），或用户当场给。
     - **有 → 直接继承**：把北极星产品 + 核心 tokens（主色/中性/语义色/圆角/密度/字体/阴影）全带进 prompt，**不再重选 style**，保证这一组概念图同源不漂移；此时风格锚点维度已满足，**不再追问风格**。
     - **无 → 临时选一个**：用产品锚点库选合理默认，并**明确提示用户**：「这只是本次概念图的临时锚点；要真正约束前端，请把视觉语言直接维护进 `docs/visual-language.md`，再把核心 token 落进工程的 `design-system.md` 与组件库主题（如 uview-plus / shadcn）——否则每次生图、每个页面都会变形。」d-prompt 只**消费**视觉语言，不是它的权威来源。
   - 缺风格锚点（没说风格、没对标产品、没给参考图，且项目无视觉锚点）→ 用知名产品对标当选项**交互式追问**或请用户给参考图。原则：只问真正缺失且影响出图的维度，**一次性问完**别来回挤牙膏，不为凑数也不设人为数量上限。Claude 用 `AskUserQuestion` 给选项（单次最多 4 问，更多则合并语义或分组）；Codex 直接列点让用户回。
   - 维度齐全 / 项目已有视觉锚点 / 用户已给风格倾向 / 用户给了参考图 → **不追问**，直接优化（参考图走「参考图驱动」：先读图提取风格再套用）。
   - 用户明确说「用我的原话 / 不要优化」→ 跳过优化，按其原意直翻直传。
3. **优化**：按 `PROMPTING.md` 的优化维度 + 产品锚点库 + 英文骨架，把需求补全为高质量 prompt。
4. **输出契约**（展示给用户过目，等确认或微调）：
   1. **中文版 prompt**（自然语言长句，便于核对语义）。
   2. **英文版 prompt**（自然语言长句，gpt-image / DALL-E 英文更稳，推荐实际使用这版）。
   3. **推荐画布尺寸 / 比例 + 理由**（按平台映射；纯信息性，用户自行在目标生图工具里设置，本 skill 不绑定任何参数）。
   4. **推断项清单**（风格/配色/组件等用户没明说、你替他补的，逐条列出可否决；走参考图驱动时额外列出从参考图提取的风格特征供用户确认）。
5. 用户确认后即结束——交付这段 prompt 即为完成，不生图、不写文件（除非用户明确要求把 prompt 存成文件）。

## 与其他 skill 的边界

- **要图片** → d-image-2（它内部也会优化 prompt 再生图）。
- **只要提示词** → 本 skill。
- 本 skill 产物可直接喂给 d-image-2 的 `--prompt`，也可粘贴到 ChatGPT 网页版 / 其他 gpt-image 接口。
