# 微信小程序规则

适用范围：微信小程序项目。后端接口、数据库、缓存、Nginx 等规则分别由对应项目或技术栈规则声明。

## 技术栈与环境

- 微信小程序（以项目 `project.config.json`、`app.json`、基础库版本和目录结构为准）

生成代码前先读取 `project.config.json`、`project.private.config.json`、`app.json`、页面/组件目录和项目已有请求、登录、状态管理写法，确认基础库版本、开发框架、分包结构和已使用插件。

## 禁止事项

- 禁止硬编码 appid、appsecret、token、密钥、API base URL；按项目配置和构建环境管理
- 禁止把 appsecret、服务端 token、支付密钥等服务端凭证放进小程序端
- 禁止日志输出手机号、身份证号、openid、unionid、session_key、token、验证码等敏感信息
- 禁止在页面 `onLoad` / `onShow` 中堆复杂业务流程或重复请求
- 禁止绕过项目既有 request 封装、鉴权处理、错误处理和响应格式
- 禁止绕过项目既有登录、授权、请求、缓存或埋点封装
- 禁止客户端直接信任价格、权限、身份、订单金额、支付状态等关键业务数据
- 禁止把 `wx.login` 返回的 code、openid、unionid 或本地缓存值直接当作已认证身份
- 禁止在本地长期保存 session_key、手机号、身份证号、完整 token 等敏感个人信息
- 禁止绕过隐私授权、用户拒绝授权、订阅消息拒绝、定位失败等失败路径
- 禁止无理由新增 npm 包、UI 框架、状态管理库、第三方库或小程序插件
- 禁止新增第三方库处理小程序原生 API 或项目已有封装能解决的问题
- 禁止主包或单个分包超过 2MB；总包大小必须按当前微信官方文档和开发者工具限制校验
- 禁止 setData 单次传输超过 256KB 或在高频场景全量传输大数组
- 禁止将非必要图片、字体等静态资源放入本地包；业务图片必须使用 CDN 地址，tabbar 图标、启动图等平台强制本地加载的资源除外
- 禁止忽略基础库兼容性、包体积、分包加载、弱网和审核影响
- 禁止让主包无控制膨胀；新增大资源、低频页面、组件库或依赖前必须评估分包和主包体积
- 禁止修改 `app.json`、分包、tabBar、插件、权限声明后不说明发布和审核影响
- 禁止把临时调试日志、mock 数据、测试账号、体验版配置提交到代码库
- 禁止用全局变量保存请求上下文、页面临时状态或未过期控制的用户数据
- 禁止在页面文件中堆复杂业务流程，必须拆到 service / model / store / util
- 禁止重复实现项目已有组件、工具函数、请求封装、登录封装或状态封装
- 禁止组件 properties / events 通过深层对象隐式耦合
- 禁止列表分页、下拉刷新、筛选、搜索出现重复请求、页码错乱和脏数据覆盖
- 禁止用未约束的动态对象在多层之间传递业务数据

## 数据、登录与授权

- 登录态、openid、unionid、用户资料、手机号授权必须遵循项目既有流程
- 需要用户授权时，必须区分静默能力、主动授权、用户拒绝和重新授权路径
- 异步数据必须处理 loading / error / empty / success 状态
- 本地缓存必须有版本、过期或清理策略

## 接口、支付与安全

- 网络请求必须有超时、错误处理和可理解的用户反馈
- 涉及支付、订单、金额、优惠、权限、身份校验时，关键判断必须由服务端完成
- 支付结果必须以后端查询或回调为准，不以客户端回调作为最终成功依据

## 页面与跳转

- 页面跳转、返回、刷新、分享入口必须考虑数据重新加载和状态恢复
- 环境配置、体验版、正式版、灰度配置按项目既有方式维护，不新增并行方案

## 文档查询

- 查询微信小程序 API、组件、基础库、支付、登录授权或审核规则时，必须使用微信官方文档、项目锁定基础库文档或源码
- 不凭记忆假设微信 API、组件行为、基础库兼容性、权限要求或审核规则
