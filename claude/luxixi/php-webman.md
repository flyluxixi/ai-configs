# PHP / Webman 规则

适用范围：PHP + Webman 项目。数据库、缓存、Web Server 的专项规则分别见 `postgresql.md`、`redis.md`、`nginx.md`。

## 技术栈与环境

- PHP（当前生产版本为准）+ Webman（常驻内存进程，非 FPM）

PHP 版本以实际部署环境为准，不在规则中写死。生成代码前先确认当前版本，使用该版本已正式发布的特性，不使用 RC / alpha 阶段的功能。

## PHP 平台能力

写入以下代码前，必须先评估平台或框架能力，不得静默绕开：

- 数据库批量处理：优先评估 SQL、批量查询、批量写入、JOIN、CTE、窗口函数；同时遵守 `postgresql.md`
- 请求路径中的连接、客户端、解析器等重量级对象：先确认不能由容器、连接池或框架生命周期管理

必须优先使用当前 PHP 版本支持的语言特性，但不为新特性而新特性。

## 禁止事项

- 禁止手动 `new PDO`；数据库连接必须由框架连接池或项目约定的数据访问层管理
- 禁止在循环内执行数据库查询；改用批量查询、`IN` 查询、JOIN、CTE 或窗口函数
- 禁止用静态属性、静态变量、单例或 `$GLOBALS` 存储请求级状态
- 禁止在请求处理中重复注册事件监听器、定时器或全局回调
- 禁止在请求处理中随意调用 `session_start`、`ini_set`、`putenv` 等会影响 Worker 后续请求的全局副作用函数
- 禁止容器单例持有 `$request`、当前用户、租户、权限、trace 上下文等请求级引用
- 禁止在请求处理中创建未归还的数据库连接、外部句柄、临时文件等非框架托管资源
- 禁止硬编码密钥、appid、appsecret、token；统一走环境变量或项目配置规范
- 禁止无说明新增 Composer 包、PHP 扩展或系统包；必须说明理由、替代方案、运行影响和回滚方式
- 禁止在中间件或请求路径中对单例 Logger 反复 `pushProcessor`
- 禁止用 `false` / `null` 表示业务失败；业务错误应抛出明确异常
- 禁止把生产环境详细错误信息直接返回给客户端
- 禁止事务失败后不回滚，或留下半写入的关联数据
- 禁止魔法数字；用常量、枚举或配置项表达业务含义

## PHP 编码规范

### 基础要求

- 所有 PHP 文件顶部必须声明 `declare(strict_types=1);`
- 命名空间必须与目录结构严格对应
- 日志轮转按项目日志体系执行；Webman 日志初始化参考 `/d-webman-log`

### 命名规范

| 场景 | 风格 | 示例 |
|---|---|---|
| 类名 | PascalCase | `OrderService` |
| 方法 / 变量 | camelCase | `createOrder`, `$userId` |
| 常量 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT` |
| 数据库字段 | snake_case | `created_at` |
| 文件 / 目录 | 与类名一致 | `OrderService.php` |

### 类型声明

优先级：`具体类型` > `?nullable` > `A|B union` > `mixed`

`mixed` 仅用于真正无法约束类型的场景（如序列化、泛型容器）。

### 代码结构

- 单个方法控制在 50 行内；超过时必须拆分，确因框架配置、映射表等无法拆分时说明原因
- 嵌套不超过 3 层；超过时必须通过提前返回、提取方法或拆分对象降低复杂度
- 单个类控制在 300 行内；超过时必须拆分职责，确需保留时说明原因

### 错误处理

- 所有 PHP 侧 HTTP / 外部服务调用必须有异常捕获
- 日志要包含足够上下文（请求 ID、用户 ID、关键参数）

## Webman 常驻内存上下文

Webman 不同于 FPM，Worker 进程常驻。请求结束后，进程内静态变量、单例、全局状态、已注册回调和未释放资源不会自动随请求销毁。

生成或修改 Webman 代码时，必须先判断代码是在 Worker 启动时执行一次，还是每个请求都会执行。生命周期判断直接决定是否存在内存泄漏、重复注册、资源残留或跨请求状态污染。

### 生命周期边界

- Worker 启动阶段适合初始化容器、路由、连接池、稳定配置和可复用客户端
- 请求处理阶段只处理当前请求的数据、权限、事务和响应
- 请求级数据不得进入静态变量、单例、全局变量或长期存活闭包
- 每请求注册监听器、定时器、processor 或全局回调，会导致重复执行或内存持续增长

### 请求状态污染

- 当前用户、租户、权限、trace id、请求参数等只在当前请求链路内传递
- 单例服务可以持有无状态依赖，不能持有当前请求对象或当前用户上下文
- 全局配置、环境变量、日志 processor 等进程级状态一旦改变，会影响后续请求

### 资源残留

- 非框架托管资源必须有明确释放点
- 事务、文件句柄、外部连接、临时文件和长生命周期回调都要按请求边界清理
- 排查 Webman 问题时优先确认是否存在跨请求复用、重复注册或未释放资源
