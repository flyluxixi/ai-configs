# Redis 规则

适用范围：缓存、限流、分布式锁、队列、异步任务、Redis 数据结构选型。

## Redis 平台能力

必须优先使用以下平台特性，不得静默绕开：

| 场景 | 应该用 | 不该绕开 |
|---|---|---|
| 计数 / 限流 | `INCR` / `DECR` / 过期时间 | 应用层变量计数 |
| 分布式锁 | `SET NX EX` + token + Lua 删除 | 先 `EXISTS` 再 `SET` |
| 批量操作 | pipeline | 循环 get / set |
| 原子组合逻辑 | Lua script | 多条命令拼竞态逻辑 |
| 消息流 | Stream | 复杂队列场景用 List 硬凑 |

## 禁止事项

- 禁止在生产环境用 `KEYS` 扫描业务 key；必须使用 `SCAN` 或维护明确索引
- 禁止业务缓存 key 无 TTL，除非明确是长期状态并有清理策略
- 禁止先 `GET` 再 `SET` 实现锁、计数或并发控制；必须使用原子命令或 Lua
- 禁止分布式锁无过期时间、无 token 校验删除；删除锁必须校验 token，不能直接 `DEL`
- 禁止把 Redis 当主数据库存放不可恢复核心数据，除非项目明确持久化、备份和恢复策略
- 禁止循环 get / set；改用 pipeline、批量命令或调整数据结构
- 禁止忽略大 key、热 key、缓存击穿、缓存雪崩风险；涉及热点 key 或高并发场景必须说明处理方式
- 禁止在日志中输出完整 token、验证码、一次性密钥等敏感值
- 禁止接口限流不使用原子计数和过期时间
- 禁止涉及金额或状态变更的 Redis key 缺少幂等性或过期策略

## 数据结构选型

| 场景 | 选择 |
|---|---|
| 字段级更新 | Hash |
| 排序 / 范围查询 | ZSet |
| 去重 / 集合运算 | Set |
| 生产者-消费者 | Stream |
| 只读整体读写 | String |

## 缓存规范

- key 命名必须稳定，包含业务前缀和必要维度，禁止冲突
- 缓存必须有过期策略；长期缓存必须说明清理方式
- 禁止将 Redis 视为唯一数据真实来源，除非项目明确持久化和恢复策略

## Redis 队列 / Stream 规范

- 所有队列任务必须实现幂等性，通过唯一业务 key 防止重复执行
- 必须定义重试策略：最大重试次数、退避间隔
- 必须配置死信处理：超出重试次数后转入死信队列并告警，禁止静默丢弃
- 任务必须设置执行超时，禁止僵尸任务长期占用 worker
- 禁止任务内嵌套分发新任务
