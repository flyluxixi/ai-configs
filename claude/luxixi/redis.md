# Redis 规则

适用范围：缓存、限流、分布式锁、队列、异步任务、Redis 数据结构选型。

## 优先使用 Redis 原子能力

遇到以下情况，必须在回复中说明为什么无法使用 Redis 能力，不得静默绕开：

- 打算手动实现计数、限流、锁，而不是用 Redis 原子命令
- 打算循环 get / set，而不是使用 pipeline
- 打算用普通读写组合实现并发控制，而不是 Lua 或原子命令

必须优先使用的平台特性：

| 场景 | 应该用 | 不该绕开 |
|---|---|---|
| 计数 / 限流 | `INCR` / `DECR` / 过期时间 | 应用层变量计数 |
| 分布式锁 | `SET NX EX` + token + Lua 删除 | 先 `exists` 再 `set` |
| 批量操作 | pipeline | 循环 get / set |
| 原子组合逻辑 | Lua script | 多条命令拼竞态逻辑 |
| 消息流 | Stream | 复杂队列场景用 List 硬凑 |

## 数据结构选型

| 场景 | 选择 |
|---|---|
| 字段级更新 | Hash |
| 排序 / 范围查询 | ZSet |
| 去重 / 集合运算 | Set |
| 生产者-消费者 | Stream |
| 只读整体读写 | String |

## 队列 / 异步任务规范

- 所有队列任务必须实现幂等性，通过唯一业务 key 防止重复执行
- 必须定义重试策略：最大重试次数、退避间隔（建议指数退避）
- 必须配置死信处理：超出重试次数后转入死信队列并告警，不能静默丢弃
- 任务必须设置执行超时，防止僵尸任务长期占用 worker
- 任务内禁止嵌套分发新任务

## 安全与日志

- 对外接口默认：参数校验 + 频率限制
- 敏感 token、验证码、一次性密钥不写入日志
- 涉及金额或状态变更的 Redis key 必须具备幂等性和过期策略
