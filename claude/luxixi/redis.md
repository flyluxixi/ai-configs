# Redis 规则

适用范围：缓存、限流、分布式锁、队列、异步任务、Redis 数据结构选型。

## 优先使用 Redis 原子能力

遇到以下情况，必须在回复中说明为什么无法使用 Redis 能力，不得静默绕开：

- 打算手动实现计数、限流、锁，而不是用 Redis 原子命令
- 打算循环 get / set，而不是使用 pipeline
- 打算用普通读写组合实现并发控制，而不是 Lua 或原子命令

必须优先使用的平台特性：

| 场景 | 应该用 | 不该绕开 |
|---|---|---|
| 计数 / 限流 | `INCR` / `DECR` / 过期时间 | 应用层变量计数 |
| 分布式锁 | `SET NX EX` + token + Lua 删除 | 先 `exists` 再 `set` |
| 批量操作 | pipeline | 循环 get / set |
| 原子组合逻辑 | Lua script | 多条命令拼竞态逻辑 |
| 消息流 | Stream | 复杂队列场景用 List 硬凑 |

## 禁止事项

- 禁止在生产环境用 `KEYS` 扫描业务 key；必须使用 `SCAN` 或维护明确索引
- 禁止业务缓存 key 无 TTL，除非明确是长期状态并有清理策略
- 禁止先 `GET` 再 `SET` 实现锁、计数或并发控制；必须使用原子命令或 Lua
- 禁止分布式锁无过期时间、无 token 校验删除；删除锁必须校验 token，不能直接 `DEL`
- 禁止把 Redis 当主数据库存放不可恢复核心数据，除非项目明确持久化、备份和恢复策略
- 禁止循环 get / set；改用 pipeline、批量命令或调整数据结构
- 禁止忽略大 key、热 key、缓存击穿、缓存雪崩风险
- 禁止在日志中输出完整 token、验证码、一次性密钥等敏感值

## 数据结构选型

| 场景 | 选择 |
|---|---|
| 字段级更新 | Hash |
| 排序 / 范围查询 | ZSet |
| 去重 / 集合运算 | Set |
| 生产者-消费者 | Stream |
| 只读整体读写 | String |

## 缓存规范

- key 命名必须稳定，包含业务前缀和必要维度，避免冲突
- 缓存必须有过期策略；长期缓存需说明清理方式
- 热点 key、缓存击穿、缓存雪崩需要说明处理方式
- 缓存数据默认不是唯一真实来源，除非项目明确 Redis 持久化和恢复策略

## Redis 队列 / Stream 规范

- 所有队列任务必须实现幂等性，通过唯一业务 key 防止重复执行
- 必须定义重试策略：最大重试次数、退避间隔（建议指数退避）
- 必须配置死信处理：超出重试次数后转入死信队列并告警，不能静默丢弃
- 任务必须设置执行超时，防止僵尸任务长期占用 worker
- 任务内禁止嵌套分发新任务

## 安全与日志

- 使用 Redis 做接口限流时，必须使用原子计数和过期时间
- 敏感 token、验证码、一次性密钥不写入日志
- 涉及金额或状态变更的 Redis key 必须具备幂等性和过期策略
