# PostgreSQL 规则

适用范围：PostgreSQL schema、迁移、查询、索引、性能排查，以及业务代码中生成或调用 PostgreSQL 查询的场景。

## 优先使用 PostgreSQL 能力

遇到以下情况，必须在回复中说明为什么无法使用 PostgreSQL 能力，不得静默绕开：

- 打算在应用层分页、聚合或排序，而不是让 SQL 一次返回结果
- 打算用多次查询组装数据，而不是 CTE / 窗口函数一次返回
- 打算循环 insert / update，而不是批量写入或 upsert

必须优先使用的平台特性：

| 应该用 | 不该绕开 |
|---|---|
| CTE | 应用层多轮拼装中间结果 |
| 窗口函数 | 应用层手动排名、分组计数 |
| JSONB | 字符串存 JSON 后应用层解析 |
| upsert（`ON CONFLICT`） | 先查再插导致竞态 |
| 全文搜索 | 简单粗暴的应用层模糊过滤 |

## 查询规范

- SQL 优先写原生查询或 Query Builder，不强制 ORM
- 禁止字符串拼接 SQL，所有查询必须参数绑定
- 禁止在循环内执行查询，改用批量查询或 `IN` 语句
- 迁移脚本写成幂等形式，可重复执行
- 软删除逻辑必须一致，定期清理孤儿记录，不允许垃圾数据长期累积

## 性能与审查

- 编写 SQL、设计 Schema、排查数据库性能时主动使用 `database-reviewer`
- ORM / QueryBuilder 生成的 SQL 也按数据库性能问题处理
- 需要跨项目对齐接口或数据结构时，先读取项目声明的接口/数据库文档，不凭记忆假设
