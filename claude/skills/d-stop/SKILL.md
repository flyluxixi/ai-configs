---
name: d-stop
description: 会话收尾 skill，更新项目状态文件。适用于 Claude Code、Codex 等 AI 编程工具。在开发工作会话结束时触发：用户说"收工"、"今天先到这"、"下次继续"、"今天就这些"、"先到这里"，或明确说"更新状态"、"记录进度"、"会话收尾"、"/d-stop"时。纯聊天、知识问答场景不触发。
---

每次开发会话结束时，把「这次做了什么、下次从哪继续」固化到文件，让下次会话可以无缝接续。

---

## Step 1：同步 STEP.md（按需）

检查项目根目录是否存在 `STEP.md` 且包含未完成的 Task：
- **有未完成 Task** → 询问用户：「本次完成了哪些 Task？」，根据回答将对应条目从 `[ ]` 标记为 `[x]`。
- **不存在或无未完成 Task** → 跳过，不提此步骤。

---

## Step 2：更新 PROJECT_STATUS.md

读取现有 `PROJECT_STATUS.md`（不存在则新建），在顶部**插入**本次记录，保留最近 3 次，更早的归入「历史记录」区块。

格式如下：

```markdown
# 项目状态

## 最后更新
[日期时间]

## 本次完成
[列出本次会话实际完成的事项]

## 下次继续
[具体的下一步行动，足够清晰让下次会话直接开始]

## 未解决
[遗留问题、待决策事项、已知 bug；无则写「无」]

## 重要上下文
[任何下次会话需要知道的背景信息；无则写「无」]

## 接口变更
[本次新增、修改、删除的 API 端点，包括路径、变更类型、请求/响应结构变化；无则写「无」]

---

## 历史记录

### [日期时间]（第 N-1 次）
**完成：** ...
**下次：** ...

### [日期时间]（第 N-2 次）
**完成：** ...
**下次：** ...
```

超过 3 条的历史记录直接删除，不保留。

---

## Step 3：更新 CLAUDE.md（按需）

如果本次会话产生了以下内容，同步更新 `CLAUDE.md`：
- 新的技术约定或规范
- 目录结构变化
- 重要架构决策
- 需要长期遵守的开发规则

无上述变化则跳过，不提此步骤。

---

## Step 4：提交状态文件到 Git

只提交以下状态文件，不暂存或提交其他工作文件：
- `PROJECT_STATUS.md`
- `STEP.md`（如本次有更新）
- `CLAUDE.md`（如本次有更新）

执行：
```bash
git add PROJECT_STATUS.md STEP.md CLAUDE.md
git commit -m "chore: update project status [日期]"
git push
```

如果上述文件不在 git 仓库中，或当前目录不是 git 仓库，跳过此步骤并告知用户。

---

## Step 5：完成确认

告知用户文件已更新，并朗读「下次继续」部分让用户确认内容准确。
