# d-prompt

把用户的图像需求，优化成一段可直接粘贴到 **ChatGPT / gpt-image-2 / DALL-E** 的高质量自然语言 prompt。**只产 prompt，不调用任何生图端点**——零外部依赖，优化全程由会话 AI 自身推理完成。

定位与边界：
- 本 skill 的产物是**提示词**，不是图片。用户拿这段 prompt 自己去任意生图工具出图。要本 skill 直接生图，用 **d-image-2**（它内部也会优化 prompt 再生图）。
- 主目标模型：**`gpt-image-2`**（含 `gpt-image-2-2026-04-21`；ChatGPT 网页版 2026 年默认底模），向下兼容 `gpt-image-1` / DALL-E 3。统一风格：**自然语言完整句式**，不堆逗号 tag 串、不带 `--ar`/`--v` 等参数（那是 Midjourney / SD 的语法，不在本 skill 范围）。
- 主用途是界面 UI 概念稿优化；非 UI 需求（人像/风景/插画/海报等）走轻量通用兜底。

## 工作流

1. **读方法论**：先读取本 skill 目录下的 `PROMPTING.md`，按其流程执行。

2. **适用判断**：按 `PROMPTING.md` 判断是 UI 需求还是通用图需求；用户说「用我的原话 / 不要优化」→ 跳过优化，按其原意直翻直传。

3. **（UI 路径，可选）项目视觉锚点**：UI 开始时做一次 0 成本检查——项目根 `docs/visual-language.md` 是否存在？或用户当场是否提供视觉规范？
   - **有 → 继承**：把北极星产品 + 核心 tokens（主色/中性/语义色/圆角/密度/字体/阴影/装饰档位/产品成熟度）全带进 prompt；后续不再追问风格，在推断项注明「已继承项目视觉锚点」。
   - **没有 → 静默跳过**进入下一步。**不要追问用户去创建该文件，不要说教让用户固化视觉语言**——`docs/visual-language.md` 是给长期维护项目的可选基础设施，d-prompt 不要求它存在。

4. **评估关键维度 + 交互式追问**：平台 / 页面类型 / 风格锚点 / **装饰档位（L0~L4）** / **产品成熟度（M0~M2）** 是否齐全？
   - 齐全 / 已继承锚点 / 已给参考图 → **不追问**直接优化（参考图走「参考图驱动」：先读图提取风格 + 装饰档位 + 成熟度档再套用）。
   - 缺风格锚点 → 用知名产品对标当选项**一次性问完**，不来回挤牙膏。
   - 装饰档位无法从「风格锚点 + 页面类型」推断 → 一并追问：克制后台（L0）/ 工具（L1）/ AI 感（L2）/ 消费 App（L3）/ 营销 Landing（L4）。
   - 产品成熟度无法从场景推断（默认 M1 Growth；用户明显说"原型 / 概念" → M0；说"企业级 / 完整中后台" → M2）→ 一并追问：早期 MVP（M0，3~5 模块）/ 成长期 SaaS（M1，6~10 模块）/ 企业级（M2，10+ 模块）。
   - 具体追问策略见 `PROMPTING.md` ②。

5. **优化**：按 `PROMPTING.md` 的优化维度（**7 个高权重维度**优先：Visual Language / Visual Decoration / **Information Architecture** / **Product Maturity** / Layout Structure / Content Realism / Design System）+ 产品锚点库 + 英文骨架，把需求补全为高质量 prompt。
   - **英文骨架尾段按维度 4 选定的装饰档位二选一**：L0~L1 用「A 强 negative」模板；L2~L4 用「B 软化 negative」模板。
   - **两套尾段共有的 "single product screen / no portfolio compilation / no multiple device screens" 约束整段照抄**——对 gpt-image-2 高频跑偏（输出作品集展示板而非真实页面）特别有效。
   - **骨架字段是 checklist，不是输出格式**：最终交付的英文 prompt 应是**流畅长句段落**，把字段融成语义自洽的叙述，**不要**输出 `Field: value.` 这种机械字段拼接句式（参考 `PROMPTING.md` 例 1 / 例 2 实际行文）。
   - 复杂页面（dashboard / 列表详情 / 表单 / 数据密集页）骨架强制包含 IA 三层；简单页面（登录 / 404 / 单一目的 onboarding）可省 IA。

6. **自检 checklist（输出给用户前必走，任一不过回到第 5 步补全）**：
   - [ ] **7 高权重维度全覆盖**：Visual Language / Visual Decoration / IA（复杂页面）/ Product Maturity / Layout Structure / Content Realism / Design System，无遗漏
   - [ ] **尾段整段在**：按维度 4 装饰档位选 A（L0~L1）或 B（L2~L4），整段照抄、一字不缺
   - [ ] **无残留占位符**：`[platform]` / `[page type]` / `[visual style]` / `[...]` 等模板占位符全部替换为实际内容
   - [ ] **Content Realism 无禁词**：无 `lorem ipsum` / `placeholder` / `sample data` / `example` / `xxx` / `1234` / `张三` / `Foo` / `Bar` / `Lorem` 等占位文案
   - [ ] **输出句式是流畅段落**：不是 `Visual decoration: X. Layout: Y. Color: Z.` 的字段拼接，是融合后的英文长句
   - [ ] **中英信息等价**：中文版与英文版**字段对齐**、不漏关键术语（设计术语保留英文或括号注释；不要求字面自然，要求字段映射可逆）
   - [ ] **推断项全列**：所有用户未明说、你替补的维度（装饰档 / 成熟度档 / IA 三层 / 配色 / 组件 / 国内锚点稳定性提示等）逐条列出可否决
   - [ ] **长度合理**：UI 200~600 英文词 / 通用图 80~250 词；超过 800 词需精简

7. **输出契约**（自检通过后展示给用户过目，等确认或微调）：
   1. **中文版 prompt**（自然语言长句，便于核对语义）。
   2. **英文版 prompt**（自然语言长句，gpt-image-2 / DALL-E 英文更稳，推荐实际使用这版）。
   3. **推荐画布尺寸 / 比例 + 理由**（按平台映射；纯信息性，用户自行在目标生图工具里设置，本 skill 不绑定任何参数）。
   4. **推断项清单**（风格 / **装饰档位** / **成熟度档** / IA 三层 / 配色 / 组件等用户没明说、你替他补的，逐条列出可否决；走参考图驱动时额外列出从参考图提取的风格特征 + 推定档位供用户确认）。
   - **载体格式（强制）**：两份 prompt 各自用 fenced code block（` ``` ` 三反引号）整段包裹，**不要用引用块 / blockquote（`>`）**——prompt 是要被用户整段复制粘贴到 ChatGPT 的，代码块才有一键复制、且复制出来不掺 `>` 等格式符号；引用块在终端 CLI 里每行带前缀，没法整段干净复制。尺寸、推断项等说明性内容正常排版即可，只有 prompt 本体进代码块。

8. **询问是否落盘**：用户确认 prompt 内容后，主动问一句「是否保存到项目 `prompts/` 目录方便后续复用？」。
   - 用户说不存 / 这次不用 / 临时用一下 → 直接结束。
   - 用户说存 / 保存 / 是 → 走「保存契约」：
     - **保存位置**：当前项目根目录下的 `prompts/`，目录不存在则新建（仅建该一层，不连带建 `prompts/ui/` 等子目录，除非用户明确指定）。
     - **文件名**：根据提示词作用用中文命名 + `.md` 后缀（如 `发布房源第二步.md` / `用户中心首页.md` / `订单列表页.md`），方便用文件名直接识别用途；用户在确认保存时若给了文件名直接用，没给则你按提示词主旨提一个让用户确认或改。
     - **文件内容**：完整保留交付契约的中英文双份 prompt + 推荐尺寸 + 推断项清单；两份 prompt 各自仍用 fenced code block 包裹（便于打开 md 一键复制 prompt 本体喂给 ChatGPT / gpt-image-2 等工具），尺寸和推断项以普通段落呈现。
     - **不要做的事**：不主动 `git add` / `git commit` / `git push`——`prompts/` 是用户的项目资产，是否提交由用户决定；写完文件告知路径即可。
   - 保存完成后即结束，不生图。

## 与其他 skill 的边界

- **要图片** → d-image-2（它内部也会优化 prompt 再生图）。
- **只要提示词** → 本 skill。
- 本 skill 产物可直接喂给 d-image-2 的 `--prompt`，也可粘贴到 ChatGPT 网页版 / 其他 gpt-image 接口。
